Introduction

REMReM (REST Mailbox for Registered Messages) is a set of tools that can be used to generate validated Eiffel messages and publish them on a RabbitMQ message bus. They can be run as micro services or as stand-alone CLI versions. For more details on the micro services and the REMReM design, see https://github.com/Ericsson/eiffel-remrem

Generate takes a partial message in JSON format, validates it and adds mandatory fields before outputting a complete, valid Eiffel message.

Installation

You can use Eiffel REMReM Generate as a micro service or a java command line application. This project is build by gradle, by executing following commands:

./gradlew build

The produced JAR file will be stored in eiffel-remrem-generate/cli/build/libs and WAR will be in eiffel-remrem-generate/service/build/libs. Then you have 2 ways to execute it: by running self executing JAR 

java -jar generate-cli.jar
or deploy it in Tomcat (put it in tomcat/webapps/ folder and start the tomcat)

Binary is released via jitPack

Download the latest version of CLI from generate-cli.jar

Download the latest version of Service from generate-service.war

Configuration

This project does not need any configuration.

Compatibility

  • JDK 8
  • Tomcat 8

For supporting latest features, Eiffel REMReM Publish should use the latest version of Eiffel REMReM Semantics.

java -jar generate-cli.jar

or deploy it in Tomcat.

Logging

REMReM Generate application logging is implemented using the logback-classic. It requires user to configure the logback.xml file.
When used without logback.xml log messages will look as below: 

%PARSER_ERROR[wEx]%PARSER_ERROR[clr] %PARSER_ERROR[clr] %PARSER_ERROR[clr] %PARSER_ERROR[clr]
            
%PARSER_ERROR[clr] %PARSER_ERROR[clr] %PARSER_ERROR[clr]

To configure the logback.xml file use -Dlogging.config=path/logback.xml
