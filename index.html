Eiffel-remrem-generate

eiffel-remrem-generate

Download this project as a .zip file Download this project as a tar.gz file

REMReM (REST Mailbox for Registered Messages) is a set of tools that can be used to generate validated Eiffel messages and publish them on a RabbitMQ message bus. They can be run as micro services or as stand-alone CLI versions. For more details on the micro services and the REMReM design, see https://github.com/Ericsson/eiffel-remrem

Generate takes a partial message in JSON format, validates it and adds mandatory fields before outputting a complete, valid Eiffel message.

Compatibility

  • JDK 8
  • Tomcat 7

The releases

Binary is relased via jitPack.

Installation

java -jar eiffel-remrem-generate-version.war or deploy it in tomcat

Usage CLI (Command Line Interface).

Examples used here are for the next generation of Eiffel events called semantics which is open source. Please follow eiffel syntax and eiffel vocabulary

java -jar generate.war -h

You passed help flag.

usage: java -jar

-d,--debug enable debug traces

-f,--content_file message content file

-h,--help show help.

-json,--json_content json content

-mp,--messaging_protocol name of messaging protocol to be used, e.g. eiffel3, semantics, semantics is used if nothing given

-r,--response_file file to store the response in, optional

-t,--message_type message type, mandatory if -f or -json is given

The message must exist in a json file for example EiffelActivityFinishedEvent.json might contain following message:
{"msgParams": { "meta": { "type": "EiffelActivityFinishedEvent", "domainId": "example.domain", "tags": ["tag1", "tag2"], "source": { "host": "host", "name": "name", "uri": "http://java.sun.com/j2se/1.3/", "serializer": { "groupId": "G", "artifactId": "A", "version": "V" } } } }, "eventParams": { "data": { "outcome": { "conclusion": "TIMED_OUT", "description": "Compilation timed out." }, "persistentLogs": [ { "name": "firstLog", "uri": "http://myHost.com/firstLog" }, { "name": "otherLog", "uri": "isbn:0-486-27557-4" } ] }, "links": { "activityExecution": "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeee1", "causes": ["cause1", "cause2"], "flowContext": "flowContext" } } }

$ java -jar generate.war -f EiffelActivityFinishedEvent.json -t eiffelactivityfinished -r output.json

Usage REST service.

Examples used here are for the next generation of Eiffel events called semantics which is open source. Please follow eiffel syntax and eiffel vocabulary

url: hostname/eiffelsemantics?msgType= use POST method body: please follow eiffel syntax and eiffel vocabulary

Testing

Example with event body passed to service:

curl -H "Content-Type: application/json" -X POST -d ' {"msgParams": { "meta": { "type": "EiffelActivityFinishedEvent", "domainId": "example.domain", "tags": ["tag1", "tag2"], "source": { "host": "host", "name": "name", "uri": "http://java.sun.com/j2se/1.3/", "serializer": { "groupId": "G", "artifactId": "A", "version": "V" } } } }, "eventParams": { "data": { "outcome": { "conclusion": "TIMED_OUT", "description": "Compilation timed out." }, "persistentLogs": [ { "name": "firstLog", "uri": "http://myHost.com/firstLog" }, { "name": "otherLog", "uri": "isbn:0-486-27557-4" } ] }, "links": { "activityExecution": "aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeee1", "causes": ["cause1", "cause2"], "flowContext": "flowContext" } } }' http://localhost:8080/eiffelsemantics?msgType=eiffelactivityfinished

Example with event body in a file body-single.json passed to service:

curl -XPOST -H "Content-Type: application/json" --data @body-single.json http://localhost:8080/eiffelsemantics?msgType=eiffelactivityfinished
